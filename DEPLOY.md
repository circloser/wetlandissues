# 인터넷에 배포하기 (Cloudflare 무료 배포 가이드)

이 서비스는 이미 **인터넷에 배포되어 운영 중**이에요. 아래 링크로 바로 접속하면 됩니다.

**이미 배포된 서비스 주소**: https://wetland-issues-mapper.singlena.workers.dev

## 새로 다시 배포해야 할 때

만약 새로 처음부터 배포해야 하거나 다른 계정으로 배포하려면, 아래 절차를 따르세요. 이 문서는 내 컴퓨터에서만 켜져 있는 서비스(로컬 실행)를, 누구나 URL 주소로 접속할 수 있는 진짜 인터넷 서비스로 올리는 방법을 설명해요. Cloudflare(클라우드플레어)라는 회사의 무료 서비스를 사용합니다.

## 준비물

- 이메일 주소 하나만 있으면 됩니다. Cloudflare 가입은 **무료**예요. 신용카드 등록도 필요 없어요.

## 배포 단계

### ① Cloudflare 가입하기

브라우저에서 [dash.cloudflare.com](https://dash.cloudflare.com) 에 접속해서 이메일로 무료 회원가입을 하세요. (이미 계정이 있다면 로그인만 하면 됩니다.)

### ② 터미널에서 로그인하기

`app` 폴더에서 터미널(PowerShell)을 열고 아래 명령을 입력하세요.

```
cd app
npx wrangler login
```

명령을 실행하면 브라우저 창이 자동으로 열려요. 그 창에서 **"허용"(Allow)** 버튼을 눌러 주세요. 이렇게 하면 터미널이 여러분의 Cloudflare 계정을 사용할 수 있도록 연결됩니다.

### ③ 데이터베이스 (이미 설정됨)

**데이터베이스는 이미 실제 Cloudflare D1에 만들어져 있고, `app/wrangler.jsonc`에 database_id가 실제 값으로 설정된 상태입니다.** 따라서 이 단계는 건너뛰어도 돼요.

새로 다시 배포해야 하는 경우에는 아래 절차를 참고하세요:

아래 명령으로 인터넷에 올라갈 새로운 데이터베이스(표)를 만들 수 있습니다.

```
npx wrangler d1 create wetland-db
```

명령을 실행하면 화면에 아래처럼 `database_id`라는 긴 값이 나와요.

```
database_id = "여기에-긴-영문숫자-조합이-나옵니다"
```

이 값을 복사해서, `app/wrangler.jsonc` 파일을 열고 아래 부분을 찾으세요.

```jsonc
"database_id": "기존값"
```

기존 값을 지우고, 그 자리에 방금 복사한 `database_id` 값을 붙여넣은 뒤 저장하세요. (따옴표는 그대로 유지합니다. 예: `"database_id": "abcd1234-...-9999"`)

> 이 파일은 메모장이나 VS Code 같은 아무 텍스트 편집기로 열어서 수정할 수 있어요.

### ④ 데이터베이스에 표 만들기 (설계도 적용)

```
npx wrangler d1 execute wetland-db --remote --file=schema.sql
```

방금 만든 인터넷용 데이터베이스 안에, 습지·뉴스 정보를 담을 표의 틀을 만드는 명령이에요. (`--remote`는 "내 컴퓨터가 아니라 인터넷에 있는 진짜 데이터베이스에 적용해라"라는 뜻이에요.)

### ⑤ 습지 초기 데이터 채워 넣기

```
npx wrangler d1 execute wetland-db --remote --file=seed.sql
```

습지 목록(내륙습지 2,986개소 또는 우리 목록으로 바꾼 데이터)을 인터넷용 데이터베이스에 채워 넣는 명령이에요.

### ⑥ 배포하기

```
npm run deploy
```

지금까지 만든 서비스 전체(지도 화면, 뉴스 수집 기능 등)를 실제로 인터넷에 올리는 명령이에요. 실행이 끝나면 화면에 접속 주소가 나옵니다.

### ⑦ 접속 주소로 들어가 보기

명령 실행 결과에 아래와 같은 형식의 주소가 나와요.

```
https://wetland-issues-mapper.<내-계정이름>.workers.dev
```

이 주소를 브라우저에 입력하면 지도 화면이 나와요. 이제 이 주소를 팀원들과 공유하면 누구나 로그인 없이 접속할 수 있어요.

### ⑧ 첫 뉴스 수집은 버튼으로 한 번 실행하기

배포 직후에는 아직 뉴스가 하나도 없을 수 있어요. 화면 위쪽 **[뉴스 수집]** 버튼을 한 번 눌러 주세요. 그러면 그 자리에서 바로 최신 뉴스를 검색해서 채워 넣어요. 이후부터는 **매일 아침 6시(한국 시각)** 에 자동으로 새 뉴스를 모으므로, 이 버튼을 매일 누를 필요는 없습니다.

## 무료 한도 안내 (걱정 안 하셔도 돼요)

- **Workers 무료 플랜**: 하루 최대 10만 번의 접속(요청)까지 무료예요. 습지센터 직원 수십~수백 명이 하루에 여러 번 들어와서 봐도 이 한도에 걸릴 일은 거의 없습니다.
- **D1(데이터베이스) 무료 한도**: 하루/월 단위로 넉넉한 무료 사용량이 제공되는데, 습지 수십 곳과 뉴스 몇백~몇천 건 정도의 소규모 내부용 데이터로는 여유 있게 사용할 수 있는 수준이에요.
- 즉, 이번 서비스처럼 **내부 직원용, 회원가입 없는 소규모 서비스**는 유료 결제 없이 계속 무료로 운영할 수 있다고 생각하시면 됩니다.

## 문제 해결 Q&A

**Q. `npx wrangler login`을 입력했는데 브라우저 로그인 창이 안 열려요.**
A. 방화벽이나 보안 프로그램이 막고 있을 수 있어요. 터미널에 나온 링크(주소)를 마우스로 복사해서 브라우저 주소창에 직접 붙여넣어 열어 보세요. 그래도 안 되면 다른 브라우저(크롬 등)를 기본 브라우저로 설정한 뒤 다시 시도해 보세요.

**Q. `database_id`를 어디에, 어떻게 넣는지 모르겠어요.**
A. `app` 폴더 안의 `wrangler.jsonc` 파일을 텍스트 편집기로 여세요. `"database_id": "local-placeholder"` 라고 적힌 줄을 찾아서, 큰따옴표 안의 `local-placeholder` 글자만 지우고 `npx wrangler d1 create wetland-db` 실행 후 나온 긴 값으로 바꿔 넣으면 됩니다. 나머지 글자(따옴표, 쉼표 등)는 건드리지 마세요.

**Q. 이미 배포된 사이트(https://wetland-issues-mapper.singlena.workers.dev)에 들어갔는데 뉴스가 적거나 없어요.**
A. 정상이에요. 처음에는 뉴스가 비어 있을 수 있어요. 화면 위쪽의 **[뉴스 수집]** 버튼을 한 번 눌러서 즉시 수집을 실행해 보세요. 버튼에 진행률이 표시되면서 뉴스가 채워집니다(예: "수집 중 1,234 / 2,986"). 수집이 끝날 때까지 기다릴 필요는 없어요 — 버튼을 누른 뒤 창을 닫아도 서버가 계속 진행합니다. 이후로는 매일 아침 자동으로 새로운 뉴스를 수집해요.

**Q. `npm run deploy` 실행 중 데이터베이스 관련 에러가 나요.**
A. ③번 단계에서 `wrangler.jsonc`의 `database_id`를 실제 값으로 바꾸지 않았거나, ④·⑤번 단계(schema.sql, seed.sql 적용)를 건너뛰었을 가능성이 커요. ③~⑤ 단계를 순서대로 다시 확인해 보세요.
