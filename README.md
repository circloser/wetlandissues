# 습지 이슈 매퍼 (Wetland Issues Mapper)

## 이 서비스는 무엇을 하나요?

전국의 습지(습지보호지역, 람사르 습지 등)와 관련된 뉴스를 컴퓨터가 매일 아침 자동으로 검색해서 모아 오고, 그 뉴스를 지도 위 해당 습지 위치에 표시해 주는 서비스예요. 직원분들은 로그인 없이 주소만 열면 바로 지도를 볼 수 있고, 지도에서 습지를 하나 클릭하면 그 습지와 관련된 뉴스 목록(제목·날짜·출처·원문 링크)이 나옵니다. 뉴스를 하나씩 보면서 "이 뉴스는 맞다(확인)", "다른 습지 얘기다(습지 수정)", "습지와 상관없다(숨김)" 중 하나로 점검할 수 있고, 이 점검 결과는 저장되어 다른 직원 화면에도 똑같이 보여요. 즉, 여러 사람이 함께 보고 함께 정리하는 공유 게시판형 지도 서비스입니다.

## 화면 구성 설명

- **지도**: 화면 왼쪽(또는 위쪽)에 대한민국 전체 지도가 나옵니다. 지도는 Leaflet이라는 무료 지도 프로그램과 OpenStreetMap이라는 무료 지도 데이터를 사용해요(네이버지도·카카오지도 같은 유료 API 키가 필요 없어요).
- **마커(습지 위치 표시)**: 지도 위에 습지마다 점(마커)이 찍혀 있어요. 그 습지에 관련 뉴스가 있으면 파란 원 안에 숫자가 표시되는데, 이 숫자가 바로 "아직 지도에 보이는 이슈(뉴스) 건수"예요. 뉴스가 없는 습지는 작은 회색 점으로만 표시됩니다.
- **뉴스 패널**: 지도 위 습지 마커를 클릭하면 화면 오른쪽에 그 습지의 이름·지역·지정유형과 함께 관련 뉴스 목록이 펼쳐져요. 각 뉴스에는 제목, 출처(언론사), 날짜, 지금 상태(미점검/확인됨) 표시가 있고, 제목을 누르면 새 창으로 원문 기사로 이동합니다.
- **점검 버튼 3종**: 뉴스 목록의 각 항목마다 아래 세 가지 버튼이 있어요.
  - **[확인]**: "이 뉴스는 이 습지 이야기가 맞다"고 표시하는 버튼이에요. 누르면 상태가 "확인됨"으로 바뀌고, 다시 누르면 "미점검"으로 되돌릴 수 있어요.
  - **[습지 수정]**: "이 뉴스는 다른 습지 얘기인 것 같다"고 판단될 때, 목록에서 올바른 습지를 골라 옮기는 버튼이에요. 옮기고 나면 원래 습지 목록에서는 사라지고, 새로 지정한 습지 쪽 건수에 반영돼요.
  - **[숨김]**: "이 뉴스는 습지와 관계없다"고 판단될 때 목록에서 완전히 숨기는 버튼이에요. 숨기면 지도의 이슈 건수에서도 빠집니다.
- **검색**: 화면 위쪽 검색창에 습지 이름을 입력하면 일치하는 습지 목록이 바로 아래에 나오고, 하나를 고르면 지도가 그 습지 위치로 자동 이동하면서 뉴스 패널도 함께 열려요.
- **기간 필터**: 시작 날짜와 끝 날짜를 지정하고 [적용]을 누르면, 그 기간에 나온 뉴스만 지도와 목록에 표시돼요. [해제]를 누르면 전체 기간으로 되돌아갑니다.
- **[뉴스 수집] 버튼**: 화면 위쪽에 있는 버튼으로, 누르면 그 자리에서 바로 최신 뉴스를 검색해서 가져와요. 원래는 매일 아침 자동으로 실행되지만, 지금 당장 새로 모으고 싶을 때 수동으로도 실행할 수 있게 만든 버튼이에요.

## 로컬 컴퓨터에서 실행해 보는 법

아래 순서대로 하나씩 따라 하면, 내 컴퓨터 안에서 이 서비스를 직접 열어볼 수 있어요. (인터넷에 배포하는 것과는 다른, "미리 보기" 단계예요.)

1. **Node.js 설치하기**
   [nodejs.org](https://nodejs.org)에 접속해서 **LTS**라고 적힌 버전을 내려받아 설치하세요. Node.js는 이 서비스 프로그램을 실행시켜 주는 기본 엔진 같은 프로그램이에요. (이미 설치되어 있다면 이 단계는 건너뛰어도 됩니다.)

2. **터미널 열기**
   윈도우에서 이 프로젝트 폴더(`wetland issues mapper`)를 열고, 그 안에서 터미널(PowerShell)을 실행하세요. 터미널은 마우스 클릭 대신 글자로 명령을 내리는 검은(또는 흰) 화면이에요.

3. **app 폴더로 이동하기**
   ```
   cd app
   ```
   실제 프로그램 코드는 `app`이라는 폴더 안에 들어 있어서, 이후 명령은 전부 이 폴더 안에서 실행해요.

4. **필요한 부품 내려받기**
   ```
   npm install
   ```
   이 서비스가 동작하는 데 필요한 프로그램 부품들(지도 라이브러리 등)을 인터넷에서 자동으로 내려받는 명령이에요. 사람이 하나하나 찾아 설치할 필요 없이 이 한 줄이면 끝나요. 처음 한 번만 실행하면 됩니다.

5. **내 컴퓨터 안에 연습용 데이터베이스 만들기**
   ```
   npm run db:local
   ```
   습지 이름·위치·뉴스를 저장할 "표(데이터베이스)"의 틀만 내 컴퓨터 안에 만드는 명령이에요. 아직 내용은 비어 있는 빈 표를 만드는 단계라고 생각하면 돼요.

6. **연습용 데이터 채워 넣기**
   ```
   npm run db:seed:local
   ```
   앞에서 만든 빈 표 안에, 미리 준비해 둔 습지 37곳의 이름·위치 정보를 채워 넣는 명령이에요. 이 데이터가 있어야 지도에 습지 점들이 나타나요.

7. **서비스 실행하기**
   ```
   npm run dev
   ```
   실제로 서비스를 켜는 명령이에요. 이 명령을 실행한 터미널 창은 서비스가 켜져 있는 동안 계속 열어 두세요(창을 닫으면 서비스도 꺼져요).

8. **브라우저에서 확인하기**
   인터넷 브라우저(크롬 등)를 열고 주소창에 아래 주소를 입력하세요.
   ```
   http://localhost:8787
   ```
   지도와 습지 마커가 보이면 성공이에요. 아직 뉴스가 하나도 없다면, 화면 위쪽 **[뉴스 수집]** 버튼을 한 번 눌러 보세요 — 잠시 후 최신 뉴스가 채워집니다.

## 뉴스가 매일 자동으로 모이는 원리

이 서비스는 배포(인터넷에 올린) 후에는 **매일 아침 6시(한국 시각 기준)** 에 자동으로 습지 관련 뉴스를 검색해서 데이터베이스에 채워 넣도록 설정되어 있어요. 사람이 매일 버튼을 누를 필요가 없다는 뜻이에요. 이 자동 실행은 Cloudflare(클라우드플레어)라는 무료 호스팅 서비스가 제공하는 "정해진 시각에 자동으로 한 번씩 실행해 주는 기능"(cron, 크론)을 사용합니다.

다만 이 자동 실행은 **인터넷에 배포된 뒤에만** 동작해요. 지금처럼 로컬 컴퓨터에서 `npm run dev`로 실행 중일 때는 자동으로 켜지지 않으니, 로컬에서 뉴스를 채우고 싶으면 화면의 **[뉴스 수집]** 버튼을 직접 눌러서 즉시 수집을 실행하면 됩니다. 배포 방법은 아래 `DEPLOY.md` 문서를 참고하세요.

## 폴더 구조 간단 설명

```
wetland issues mapper/
├── README.md                 이 문서 (사용법 안내)
├── DEPLOY.md                 인터넷에 배포하는 방법 안내
├── 습지_이슈_wetland_issues_mapper_.md   원래 기획 메모
└── app/                      실제 서비스 프로그램이 들어 있는 폴더
    ├── package.json          이 프로그램을 실행하는 명령어 목록
    ├── wrangler.jsonc        Cloudflare 배포·설정 파일 (데이터베이스 연결 정보 등)
    ├── schema.sql             데이터베이스 표의 틀(설계도)
    ├── seed.sql               습지 37곳의 초기 데이터(이름·위치 등)
    ├── src/                   서버 쪽 코드 (뉴스 수집, API 등)
    ├── public/                브라우저에 보이는 화면 코드 (지도, 버튼 등)
    └── test/                  자동 테스트 코드
```

## 습지 목록을 우리 데이터로 바꾸기

### 지금 들어있는 습지 37곳은 임시 데이터예요

현재 지도에 표시되는 습지 37곳은 국립생태원·환경부·람사르협약 등에서 공개한 **습지보호지역·람사르 등록 습지 목록을 참고해 만든 임시 데이터**입니다. 실제 업무에 사용하려면 습지센터가 보유한 정식 습지 목록으로 바꿔야 해요.

### 교체하는 방법

습지 목록은 `app/seed.sql` 파일 안에 아래와 같은 형식으로 한 줄씩 들어 있어요.

```sql
('습지 이름', 위도, 경도, '지역', '지정유형'),
```

예를 들어 `('우포늪', 35.5550, 128.4180, '경남 창녕', '람사르'),` 처럼 **이름 / 위도 / 경도 / 지역 / 지정유형** 다섯 가지 정보가 한 습지당 한 줄씩 필요해요.

만약 습지센터가 가지고 있는 목록이 엑셀 파일이라면, 개발을 도와주는 담당자(또는 AI 도우미)에게 그 엑셀 파일을 주면서 "이 목록으로 `seed.sql` 파일을 만들어 줘"라고 요청하시면 돼요. 엑셀의 열(이름, 위도, 경도, 지역, 지정유형)만 위 형식에 맞게 옮겨 적는 작업이라 어렵지 않게 바꿀 수 있습니다.

파일을 새로 만들었다면, 아래 명령으로 반영합니다.

- **로컬(내 컴퓨터)에 반영**: `app` 폴더에서
  ```
  npm run db:seed:local
  ```
- **이미 인터넷에 배포된 서비스에 반영**: `app` 폴더에서
  ```
  npx wrangler d1 execute wetland-db --remote --file=seed.sql
  ```
  (배포와 관련된 자세한 내용은 `DEPLOY.md`를 참고하세요.)

### 위도·경도(좌표)가 없는 경우

지도에 점을 찍으려면 각 습지의 **위도·경도(좌표)** 가 반드시 필요해요. 만약 가지고 있는 목록에 이름만 있고 좌표가 없다면, 먼저 좌표를 구하는 작업이 필요합니다. 목록(이름, 지역 등)이 준비되면 개발 도우미(AI)에게 "이 목록으로 seed.sql 만들어 줘, 좌표는 지역명 기준으로 찾아 줘"라고 요청하시면 좌표를 찾아서 형식에 맞게 정리해 드릴 수 있어요.
